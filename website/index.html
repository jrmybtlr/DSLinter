Design-system governance

Lint your components for design-system health

DSLinter scans JSX, TSX, and Vue repos for component usage, accessibility signals, hardcoded tokens, and governance scores — in the terminal, CI, or a live dashboard.

Get started View on GitHub
$ npm install -D dslinter

Everything you need to govern UI code

Storybook-style visibility without story files. Powered by Oxc for fast JS/TS/JSX parsing.

  • Usage counts

    Discover how often each component is used across JSX and Vue templates with prop hints.

  • Accessibility checks

    Catch missing alt text, unnamed buttons, and other a11y issues scoped to each component module.

  • Token governance

    Flag hardcoded colors, Tailwind arbitrary values, and unused CSS custom properties.

  • CI and dashboard

    Run one-shot reports in CI with JSON output, or watch locally with the live governance dashboard.

Quick start

Install the npm package, then run the CLI from your project root. Prebuilt native binaries ship with the package — no Rust toolchain required.

  • Local dev with watch mode and dashboard UI
  • One-shot JSON reports for CI pipelines
  • Optional .dslinter.json config
Terminal
npm install -D dslinter
npx dslinter init
npx dslinter
npx dslinter --report /path/to/repo --json
npx dslinter --report --output public/dslinter-report.json
